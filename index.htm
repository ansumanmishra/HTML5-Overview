Store Dynamic Content

Cookies - Drawbacks

  • Allow up to only 4 KB of data storage
  • Cookies are sent with every HTTP request
  • Each domain stores all its cookies in a single string, which can make parsing data difficult
  • SQL injection can be performed from a cookie

This is where HTML5 storage comes in

  • Local Storage
  • Session Storage

Local Storage & Session Storage

Pros
  • ~5MB storage per domain
  • Support by most modern browsers
  • Persistent data that is stored directly in the browser
  • Is not sent with every HTTP request
Cons
  • Not supported in IE8 and few older browsers
  • Data Can't be accessed by server

sessionStorage

window.sessionStorage
  • Same as localStorage but...
  • Lasts as long as browser is open
  • Opening page in new window or tab starts new session
  • Great for temporal data (e.g. form progress)

Web Storage APIs only store strings!

Solution:

var userstr = JSON.stringify({
  user: 'john',
  id: 10
});

localStorage.setItem('user', userstr);

var user = JSON.parse(localStorage.getItem('user'));

}, false);

App Cache Manifest Sections

CACHE:
# Cached files go here

NETWORK:
# Resources that require online access
# Can match by prefix or wildcard: /api/

FALLBACK:
# Substitutions to serve when offline
# Can match by prefix

App Cache Resource Loading

  • 1. NETWORK
    Fetch from network
  • 2. CACHE
    Load from cache
  • 3. FALLBACK
    a) Try to fetch from network
    b) If failure, load cache resource
  • 4. If '*' in a NETWORK section
    Try to fetch from network

Canvas

Canvas

Canvas gives an easy and powerful way to draw graphics using JavaScript

           <canvas id="draw-square" width="300" height="300"></canvas>
           
           <script>
             function draw () {
                var canvas = document.getElementById('draw-square');
                if (canvas.getContext) {
                  var context = canvas.getContext('2d');

                    context.fillStyle = 'blue';
                    context.fillRect (12,12,96,96);

                    context.fillStyle = "#FF1C0A";
                    context.beginPath();
                    context.arc(100, 100, 90, 0, Math.PI*2, true);
                    context.closePath();
                    context.fill();
                } else {
                  alert("Your browser does not support Canvas")
                }
              }

            draw();
           </script>

Web Worker

Web Worker

Any script that you want to execute in the background is a good candidate to run as a web worker

Usage : 

Intense mathematical/scientific computations 

Big amount of data you want to process retreived from an AJAX request

Background search in dictionaries to help the user typing

Limitations : 

Can't access DOM objects 

Can't access global/local Javascript variables or functions

Web Worker - Example

            <div id="result"></div>
            <a href="#" onclick="startWorker(); return false;">Start Worker</a>
            <a href="#" onclick="stopWorker(); return false;">Stop Worker</a>
          
          <script type="text/javascript">
          var w;

          function startWorker() {
           // Checking if the browser supports web workers. Another way to do this by - if (window.Worker) { Do Some Stuffs } else { alert('Not supported')}
           if(typeof(Worker)!=="undefined") {
            w = new Worker("count.js");
            w.postMessage();

            w.onmessage = function(event){
             document.getElementById("result").innerHTML=event.data;
            };
            worker.onerror = function (event) {
                   console.log(event.message, event);
                };
           }
           else { alert("Sorry!! Your browser does not support Web worker"); }
          }
          function stopWorker(){
           w.terminate();
          }
          </script>
